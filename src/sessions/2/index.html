Stream Magic

Streams allow us to

  • operate on data chunk by chunk
  • build around a common abstraction

How To Streams

  • Readable
    • produces data: you can pipe FROM it
    • readable.pipe(A)
  • Writable
    • consumes data: you can pipe TO it
    • A.pipe(writable)
  • Transform
    • consumes data, producing transformed data
    • A.pipe(transform).pipe(B)

Server - Broadcast State

// server.js
// ...
var websocket = require('websocket-stream')

var wsServer = websocket.createServer({
  server: httpServer,
}, wsHandler)

var position = [0.5, 0.5]

function wsHandler (stream) {
  var state = JSON.stringify(position)
  stream.write(state)
  stream.pipe(process.stdout)
}

Browser - Listen For State

// browser.js

var websocket = require('websocket-stream')
var through = require('through2')

var stream = websocket("/")
stream.pipe(through(function (state) {
  var position = JSON.parse(state)
  console.log(position)
}))

Browser - Render State

// browser.js

var websocket = require('websocket-stream')
var through = require('through2')
var dom = require('domquery')

var stream = websocket("/")
stream.pipe(through(function (state) {
  var position = JSON.parse(state)

  dom('main').add(position)
}))

Event Magic

Events allow us to do something when something else happens.

We can listen to event emitters (objects that "emit events") with emitter.on(eventName, callback).

Duplex Stream Magic

  • Duplex A.pipe(duplex).pipe(A)
